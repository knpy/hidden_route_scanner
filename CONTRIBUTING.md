# 開発ルール

## コンフリクト防止のベストプラクティス

⚠️ **重要**: 新しいブランチを作成する前に、必ず以下を実行してください

```bash
# 1. mainブランチに移動
git checkout main

# 2. 最新のmainを取得
git pull origin main

# 3. 新しいブランチを作成
git checkout -b feature/your-feature-name
```

**理由**: 古い `main` から派生したブランチでPRを作成すると、他のPRが先にマージされた場合にコンフリクトが発生します。

## プルリクエスト (PR)
- PRを作成した後は、マージされているかを確認してください（手動でマージされることがあります）。
- PRがマージされたら、**次のブランチを作る前に必ず上記の手順で `main` を更新してください**。

## 新機能開発
- 新たな機能を作るときは、新しいブランチを作成してください。
- ブランチを作成する際は、必ずリモートリポジトリの最新の状態を `pull` してから行ってください。
